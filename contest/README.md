## 说明

生成一个巨大的makefile，完成所有的编译、评测功能。这么做的好处有很多：

1. 在多核心的处理器上可以直接make -jXX并行评测。
2. 当有修改时（例如新的选手到来）不需要重新测试。
3. 修改数据时不需要全部重新评测。

以下说明Makefile用到的工具。

### compile

给定源程序和正确的后缀名，用正确的工具生成可执行文件。

### judge

给定可执行文件、输入文件、输出文件，在沙盒中运行之，并给出评测结果。评测结果分为若干行：

1. 对结果的说明（正确/错误/超时……）。
2. 该测试点得分的比例（0-1之间的浮点数）。
3. 之后是对评测结果的说明（可选）。

## 竞赛文件格式

竞赛是YAML格式的文件。由“---”分开每个题目。

每个题目都有一些必备的域：

1. name，题目的长描述
2. abbrv，题目的短英文描述，只包含字母和数字
3. path，题目数据存放的位置
4. allowed_file，接受的源文件名
5. time_limit，时限，为测试点默认的时限(ms)
6. memory_limit，内存限制，为测试点默认的内存限制(MB)
7. score，每个测试点的默认满分
8. testcases，描述每个测试数据

题目可选的域：

1. spj，用来评测的程序

每个测试数据有以下必备的域：
1. input，输入文件名
2. output，正确的输出文件名

测试点可选的域：
2. time_limit，该测试点的时间限制(ms)
3. memory_limit，该测试点的内存限制(MB)
4. score，该测试点的满分
